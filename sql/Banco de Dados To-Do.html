🔹 PASSO 6 — Criando a tabela tasks (com explicação real)

Antes do código, pensa comigo:

Uma tarefa precisa de:

  • id → identificador

  • title → título da tarefa

  • description → descrição

  • is_done → feita ou não (booleano)

  • user_id → quem é o dono da tarefa

👉 Esse último é o mais importante de todos.

🔥 Aqui está o ponto que você não pode ignorar

Se você NÃO colocar user_id:

  • seu app não sabe de quem é a tarefa

  • qualquer usuário pode ver tudo

  • você destrói a lógica do sistema

✅ Agora o código (lê junto, não só copia)

SQL
CREATE TABLE tasks (
    id INT PRIMARY KEY IDENTITY(1,1),
    title VARCHAR(150) NOT NULL,
    description VARCHAR(255),
    is_done BIT DEFAULT 0,
    user_id INT NOT NULL,
    created_at DATETIME DEFAULT GETDATE(),

    FOREIGN KEY (user_id) REFERENCES users(id)
);

🔍 Explicação sem enrolação

  • BIT → tipo booleano (0 = falso, 1 = verdadeiro)

  • DEFAULT 0 → tarefa começa como “não feita”

  • user_id → guarda o dono da tarefa

  • FOREIGN KEY → obriga que o usuário exista

⚠️ ERRO QUE VOCÊ PROVAVELMENTE FARIA

Se eu não te mostrasse:

❌ Criar user_id sem FOREIGN KEY
→ banco não protege relação

❌ Não colocar NOT NULL
→ tarefa sem dono (lixo de dado)
